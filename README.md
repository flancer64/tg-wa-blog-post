# tg-wa-blog-post

Веб‑сервис для публикации записей блога в телеграм‑канал через бота.

## Подготовка

1. Установите зависимости:
   ```bash
   npm install
   ```
2. Создайте файл `.env` на основе `.env.example` и задайте `BOT_TOKEN`.

## Варианты запуска

### Штатный запуск

Штатный режим читает последнее сообщение из исходного ru-канала в Telegram, переводит его и публикует результаты в en и es каналы.

Запуск через `npm`:

```bash
npm start
```

Прямой запуск через Node.js:

```bash
node bin/bootstrap.mjs
```

### Аварийный ручной запуск из файла

Аварийный режим используется, когда сообщение из ru-канала не было обработано автоматически, и нужно вручную запустить перевод и публикацию без повторного чтения из Telegram.

Требования к входному файлу:

- файл должен содержать только plain text тело одной ru-публикации;
- структурированные метаданные не поддерживаются;
- файл не должен быть пустым.

Запуск:

```bash
node bin/bootstrap.mjs --source-file ./var/manual/post.txt
```

В этом режиме приложение:

- читает текст из указанного файла;
- переводит его для en и es;
- публикует переводы в целевые каналы;
- сохраняет агрегат с отрицательным `ru_message_id` формата `-YYYYMMDDHHMM`.
