# CLI Contract

Path: `./ctx/docs/code/cli.md`

## Назначение

Документ фиксирует инженерный контракт CLI-интерфейса Telegram Translation Publisher.

## Модель интерфейса

CLI предоставляет единственный запуск run-cycle без командного дерева и диспетчеризации. Интерфейс допускает аварийный ручной источник ru-текста через параметр `--source-file`, не образуя отдельной команды или второго production entrypoint.

## Аргументы

По умолчанию bootstrap передаёт в `App.run()` массив `process.argv.slice(2)` для штатного запуска чтения из Telegram. CLI-контракт также допускает параметр `--source-file <path>`, который переключает источник ru-текста на локальный plain text файл для аварийного ручного режима. Иные функциональные CLI-флаги не являются частью текущего контракта.

## Поведение запуска

Один CLI-запуск соответствует одному run-cycle и завершается возвратом `exitCode`, определённого `App.run()`. Запуск с `--source-file ./var/manual/post.txt` остаётся тем же CLI-процессом и отличается только способом получения исходного ru-текста.

## Связь с логированием

CLI не управляет уровнями логирования через аргументы.

## Итог

CLI-контракт закрепляет минимальный интерфейс запуска с одним optional-флагом аварийного ручного чтения из файла и исключает расширение модели команд в рамках текущего продукта.
