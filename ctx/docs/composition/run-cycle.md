# Run Cycle Composition

Path: `./ctx/docs/composition/run-cycle.md`

## Назначение

Документ фиксирует композиционный контракт одного run-cycle Telegram Translation Publisher.

## Последовательность фаз

Один CLI-запуск содержит не более одного run-cycle со следующими фазами:

1. Определение источника ru-публикации: Telegram по умолчанию или local source-file при наличии `--source-file`.
2. Получение последнего ru-сообщения из Telegram либо чтение plain text содержимого source-file.
3. В штатном режиме при отсутствии сообщения завершение с `exitCode = 0`.
4. В аварийном ручном режиме при невозможности чтения файла или при пустом содержимом завершение с `exitCode = 1`.
5. Проверка существования агрегата по `ru_message_id`.
6. При наличии агрегата завершение без побочных действий с `exitCode = 0`.
7. Определение типа сообщения и выделение `text`, `caption` и media-части.
8. Выполнение ветки en: для `text` перевод и публикация, для media прямое воспроизведение, для caption перевод и публикация с media.
9. Выполнение ветки es: для `text` перевод и публикация, для media прямое воспроизведение, для caption перевод и публикация с media.
10. Формирование агрегата.
11. Атомарная фиксация агрегата в storage.
12. Возврат итогового `exitCode`.

## Параллельность веток

Ветки en и es независимы и могут выполняться параллельно; порядок их завершения не фиксируется.

## Правило статуса и `exitCode`

`status = success` только при успешном завершении обеих веток публикации, иначе `status = failure`. При `failure` run-cycle завершает CLI-запуск с `exitCode = 1`.

## Идемпотентность

Наличие агрегата для `ru_message_id` блокирует повторный run-cycle и повторную публикацию. В аварийном ручном режиме идемпотентность обеспечивается отрицательным `ru_message_id` формата `-YYYYMMDDHHMM`.

## Итог

Run-cycle является детерминированной фазовой композицией с бинарным статусом агрегата и явными точками раннего завершения.
