# Продуктовый контекст (`./ctx/docs/product/`)

- Path: `./ctx/docs/product/AGENTS.md`
- Version: `20260212`

## Назначение

Каталог `ctx/docs/product/` фиксирует продуктовый смысл системы: назначение, границы предметной области, инварианты и ожидаемый функциональный результат. Уровень описывает, что представляет собой продукт и какие свойства остаются неизменными при реализации, не переходя к архитектурным или процедурным деталям.

## Карта уровня

- `AGENTS.md` — текущий документ уровня `product/`, задающий его границы, требования и навигацию.
- `overview.md` — декларативное описание назначения продукта, его предметной области, смысловой модели, инвариантов и границ применения.

## Границы уровня

Уровень `product/` ограничен описанием продуктовых смыслов и не включает архитектурные решения, инженерные регламенты, операционные процедуры и поведенческие инструкции агента. Внутри уровня утверждения должны оставаться декларативными, непротиворечивыми и достаточными для понимания назначения системы без обращения к другим ветвям документации.

## Требования

1. Каждый документ уровня обязан сохранять декларативный стиль и не подменять описание продукта инструкциями по реализации.
2. Новые документы уровня должны расширять продуктовую модель без дублирования утверждений, уже зафиксированных в `overview.md`.
3. При изменении состава каталога `ctx/docs/product/` раздел «Карта уровня» в `ctx/docs/product/AGENTS.md` обновляется синхронно.
4. Текст внутри абзацев оформляется без ручных переносов строк.

## Итог

`ctx/docs/product/AGENTS.md` определяет локальные правила продуктового уровня и фиксирует пространство для описания назначения и инвариантов системы.
