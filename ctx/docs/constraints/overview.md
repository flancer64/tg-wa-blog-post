# Ограничения реализации

Path: `./ctx/docs/constraints/overview.md`

## Назначение уровня

Документ фиксирует обязательные ограничения Telegram Translation Publisher без повторного описания архитектурной формы, композиции фаз и инженерных механизмов.

## Продуктовые ограничения

Система обслуживает фиксированную трёхканальную модель `ru → en, es` и не расширяет языковой состав.

## Ограничения обработки

Обрабатывается только текстовый контент; медиа и вложения не входят в контракт.

## Ограничения запуска

Поддерживается только один CLI-запуск одного run-cycle без дерева команд, dry-run и альтернативных режимов.

## Ограничения статуса

`status` агрегата имеет только бинарные значения `success` и `failure`; частичный результат относится к `failure`.

## Ограничения хранения

Файловое storage остаётся единственным источником истины для факта обработки `ru_message_id`; модель хранения остаётся append-only с атомарной фиксацией.

## Ограничения согласованности

После публикаций rollback не выполняется; при неудачной фиксации агрегата повторная публикация в следующем запуске допускается.

## Ограничения архитектуры кода

DI-модель, storage-модель и CLI-модель являются фиксированными и не заменяются альтернативными механизмами в рамках текущего продукта.

## Итог

Ограничения уровня задают неизменяемые рамки продукта и реализации, в которых все остальные уровни документации остаются согласованными.
