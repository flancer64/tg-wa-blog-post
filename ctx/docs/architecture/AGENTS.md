# Архитектурный контекст (`./ctx/docs/architecture/`)

- Path: `./ctx/docs/architecture/AGENTS.md`
- Version: `20260212`

## Назначение

Каталог `ctx/docs/architecture/` фиксирует архитектурную форму системы: структурные контуры, доменную модель агрегата, внешние интерфейсы и инварианты файлового хранения. Уровень описывает что составляет архитектуру и где проходят её границы, не переходя к продуктовым смыслам, организационным режимам агента и процедурной реализации.

## Карта уровня

- `AGENTS.md` — текущий документ уровня `architecture/`, задающий границы, требования и навигацию по архитектурным материалам.
- `domain-model.md` — декларативная модель агрегата публикации: идентичность, состав, целостность и неизменяемость.
- `external-interfaces.md` — архитектурные контракты взаимодействия с Telegram Bot API и LLM API.
- `overview.md` — целостное описание архитектурной формы: контуры, жизненный цикл запуска и внешние границы.
- `storage-model.md` — модель файловой фиксации агрегатов, принципы адресации и инварианты записи.

## Границы уровня

Уровень `architecture/` ограничен структурой системы и архитектурными инвариантами. Документы этого каталога не должны подменять инженерные регламенты реализации, не должны дублировать продуктовые определения уровня `product/` и не должны включать инструкции по поведению агента.

## Требования

1. Новые документы уровня добавляются только для расширения архитектурной модели и не должны переносить в `architecture/` регламенты реализации.
2. При изменении состава каталога раздел «Карта уровня» обновляется синхронно и остаётся навигационно точным.
3. Текст внутри абзацев оформляется без ручных переносов строк.

## Итог

`ctx/docs/architecture/AGENTS.md` определяет локальные границы архитектурного уровня и фиксирует набор документов, описывающих форму, контракты и инварианты архитектуры Telegram Translation Publisher.
