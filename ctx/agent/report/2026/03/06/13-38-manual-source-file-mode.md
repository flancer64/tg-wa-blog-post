# Отчёт итерации

Path: `ctx/agent/report/2026/03/06/13-38-manual-source-file-mode.md`

## Резюме изменений

Зафиксировано расширение когнитивного контекста проекта для аварийного ручного режима запуска публикации из локального plain text файла через CLI-параметр `--source-file`. Режим описан как компенсационный сценарий для пропущенной обработки сообщения из ru-канала без изменения статуса Telegram-канала как основного источника истины.

## Детали работ

Уточнён продуктовый контекст в `ctx/docs/product/overview.md`: добавлено допущение аварийного ручного запуска из файла при сохранении всех продуктовых инвариантов репликации.

Уточнён архитектурный контекст в `ctx/docs/architecture/glossary.md`, `ctx/docs/architecture/domain-model.md`, `ctx/docs/architecture/overview.md` и `ctx/docs/architecture/external-interfaces.md`: добавлены определение аварийного ручного режима, модель локального source-file как внешнего источника текста и правило генерации отрицательного `ru_message_id` формата `-YYYYMMDDHHMM`.

Уточнён композиционный и кодовый контекст в `ctx/docs/composition/overview.md`, `ctx/docs/composition/run-cycle.md`, `ctx/docs/code/cli.md` и `ctx/docs/code/external/telegram.md`: зафиксированы взаимоисключающие источники ru-текста в одном CLI-запуске, поведение `--source-file`, условия завершения run-cycle и исключение чтения из Telegram в аварийном ручном режиме.

## Результаты

Когнитивный контекст проекта теперь допускает запуск перевода и публикации поста без обращения к исходному Telegram-каналу при явной передаче `--source-file <path>`.

Нормативный формат входного файла зафиксирован как plain text тело одной ru-публикации без структурированных метаданных.

Правило доменной идентичности для аварийного режима зафиксировано как отрицательный `ru_message_id` формата `-YYYYMMDDHHMM`.

Тесты не выполнялись, поскольку итерация ограничена документацией и не включает изменения продуктового кода.
